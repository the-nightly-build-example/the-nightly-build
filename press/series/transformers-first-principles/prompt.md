# Transformers from First Principles

Build an accurate mental model of transformer language models without asking the
reader to program. Each lesson isolates one component or decision, then shows
how it changes the system that came before it. Use a tiny sentence, a table of
tokens, or a hand-worked calculation when it makes the machinery visible. Avoid
code and avoid symbolic derivations that obscure the mechanism.

Distinguish architecture, training objective, training data, inference-time
decoding, and the surrounding product system. A transformer is neither a mind
nor magic autocomplete. State exactly what a component computes, what signal it
receives, and what limitation remains.
